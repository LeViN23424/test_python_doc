Чек-лист регрессии для консольного модуля задач

Этот чек-лист предназначен для быстрой проверки основных функций перед релизом

1. Запустить программу без аргументов — убедиться, что интерфейс доступен и ошибок нет.
2. Создать новую задачу с валидными данными — проверить появление задачи в списке.
3. Изменить статус существующей задачи — убедиться, что статус обновился и сохранился.
4. Сохранить состояние задач в файл JSON — проверить отсутствие ошибок.
5. Завершить программу, запустить повторно и загрузить задачи — убедиться, что данные не потерялись.
6. Выполнить поиск по ключевому слову — проверить корректность фильтрации.
7. Попытаться создать задачу с пустым названием — убедиться, что программа корректно обрабатывает ошибку (например, выводит предупреждение).
8. Попытаться создать задачу с некорректным статусом — убедиться, что программа не принимает неверные значения.
