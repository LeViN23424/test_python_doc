Тест-кейсы для консольного модуля задач

TC-01: Создание задачи с валидными данными (Счастливый путь)
Название: Создание задачи с валидными полями
Предусловия: Файл данных отсутствует или пустой
Шаги:
1. Запустить программу из терминала
2. Ввести команду создания задачи: add Отчёт Сдать ЛР
3. Ввести команду просмотра списка: list
 Ожидаемый результат: В списке отображается задача с названием «Отчёт», описанием «Сдать ЛР», уникальным идентификатором и статусом по умолчанию (например, new).
 Постусловия: В файле данных появилась запись о новой задаче
TC-02: Создание задачи с пустым названием (Ошибочный сценарий)
Название: Создание задачи с пустым названием
Предусловия: Программа запущена, список задач доступен
Шаги:
1. Ввести команду создания задачи с пустым названием: add "" Описание
2. Ввести команду просмотра списка: list
 Ожидаемый результат: Программа выводит сообщение об ошибке (например, "Название задачи не может быть пустым") задача не добавляется в список и не сохраняется в файл
 Постусловия: Список задач и файл данных не изменились
